ABs don't connect to your systems through APIs. They log into the same portals, dashboards, and inboxes your buyers and dispatchers use. No API contracts. No IT integration projects.

If a person can log into it, an AB can operate it.

## What an AB does when it gets a task

1. Opens the right surface — SAP, carrier portal, TMS, inbox, or supplier portal
2. Logs in, including 2FA
3. Navigates to the right page or view
4. Reads data, fills forms, pulls files, drafts memos
5. Returns the result in your channel with citations and an audit link

## Why no API integrations

Procurement teams can't spend quarters on API contracts and IT integration tickets. Computer use means the Brain enrolls in a surface the same day onboarding starts. If the surface changes its UI, the AB adapts. Legacy systems, portals with no API, carrier dashboards — all work the same way.

<Warning>
  ABs need network access to the surfaces they operate. In air-gapped deployments, they can only reach surfaces available on the same local network. Configure surface access during [onboarding](/setup-wizard).
</Warning>

<Tip>
  If a portal uses 2FA, ensure the receiving phone number or email is accessible while the AB is running. The onboarding wizard handles 2FA enrollment for each surface.
</Tip>
